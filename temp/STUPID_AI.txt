Summary of What We Know

1. **Key Format**:
   - The `key` parameter in `didkit.issueCredential` expects a JSON string representing a JSON Web Key (JWK).
   - The JWK must contain both public and private components to sign the credential.
   - Example JWK format:
     ```json
     {
       "kty": "OKP",
       "crv": "Ed25519",
       "x": "...",
       "d": "..."
     }
     ```

2. **Verification Method**:
   - The `verificationMethod` must be a URI that points to the public key needed to verify the credential's signature.
   - It must belong to the DID identified as the issuer in the credential.
   - Use `didkit.keyToVerificationMethod("key", key)` to generate the correct URI from the JWK.

3. **@context**:
   - The `@context` field is mandatory and defines the semantics of the terms used in the credential.
   - It must include the base W3C Verifiable Credentials context: `"https://www.w3.org/2018/credentials/v1"`.
   - If the credential uses custom fields, include additional URIs in the `@context` array.

4. **Current Issue**:
   - The `key` is not being passed correctly to `didkit.issueCredential`.
   - The `verificationMethod` and `@context` are not being correctly formatted.
